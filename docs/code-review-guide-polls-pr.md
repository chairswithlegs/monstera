## 2. Database migration

**Files:** `internal/store/migrations/000037_create_polls.{up,down}.sql`

- [ ] **Naming**  
  - Migration number follows existing sequence (e.g. 000037).

- [ ] **Schema**  
  - **polls:** `id` (PK), `status_id` (FK to statuses), `multiple`, `hide_totals`, `votes_count`, `voters_count`, `expires_at` (timestamptz nullable). Sensible defaults (e.g. 0 for counts).
  - **poll_options:** `id` (PK), `poll_id` (FK), `position` (smallint), `title`, `votes_count`.
  - **poll_votes:** `id` (PK), `poll_id` (FK), `account_id` (FK), `choices` (e.g. smallint[] or equivalent), `created_at`. UNIQUE on `(poll_id, account_id)`.

- [ ] **Indexes**  
  - At least: `polls(status_id)`, `poll_votes(poll_id, account_id)` (or unique constraint covering that).

- [ ] **Down migration**  
  - Drops tables in safe order (e.g. poll_votes â†’ poll_options â†’ polls).

## 3. Store layer

**Files:**  
- `internal/store/store.go` (new methods)

- `internal/store/postgres/queries/polls.sql`

### 3.1 Store interface and types

- [ ] **Interface**  
  - New methods: `CreatePoll`, `GetPollByID`, `GetPollByStatusID`, `CreatePollVote`, and optionally `GetPollVoteByAccountAndPoll` if used. Signatures take `context.Context` and domain-friendly input/return types.

- [ ] **Input/result types**

### 3.2 SQL and Postgres implementation

- [ ] **Queries**  
  - Defined in `.sql` under `internal/store/postgres/queries/`; use sqlc-compatible style. No raw SQL in Go for these operations (use sqlc-generated code).

- [ ] **Error handling**  
  - `pgx.ErrNoRows` â†’ `domain.ErrNotFound`.  
  - Unique violation (e.g. `23505`) â†’ `domain.ErrConflict`.  
  - Other errors wrapped with `%w` and context (e.g. `"GetPollByID(%s): %w", id, err`). Never use `%v` for wrapping.

- [ ] **Transactions**  
  - `CreatePollVote` (and any â€śinsert vote + update countsâ€ť) runs in a transaction so the result is consistent.

- [ ] **Conventions**  
  - No logging in store; no `net/http`. Use `pgxpool`; donâ€™t hold connections outside request scope.

## 4. Service layer

**Files:**  
- `internal/service/status_service.go` (PollInput, CreateResult.Poll, poll creation in tx)  
- `internal/service/poll_service.go` (new)

### 4.1 Status service (create status with poll)

- [ ] **CreateWithContentInput**  
  - Has `Poll *PollInput` (nil = no poll). `PollInput`: options, expires_in (seconds), multiple, hide_totals.

- [ ] **CreateResult**  
  - Includes `Poll *domain.Poll` when a poll was created.

- [ ] **Transaction**  
  - Poll and options created in the same transaction as the status (e.g. inside `createStatusWithContentTx`). No separate commit for the poll.

- [ ] **Errors**  
  - Returns domain errors; wraps with `%w` and business context. No HTTP, no logging before return.

### 4.2 Poll service

- [ ] **Interface**  
  - e.g. `GetByID(ctx, pollID, viewerAccountID *string)` and `Vote(ctx, VotePollInput)`. Return type carries poll + voted + own_votes (or equivalent).

- [ ] **Vote validation**  
  - Poll not expired; voter hasnâ€™t already voted; choices in range; if multiple choices, poll allows multiple.  
  - Already voted â†’ `domain.ErrConflict`. Expired â†’ e.g. `domain.ErrGone` or agreed sentinel.

- [ ] **Dependencies**  
  - Service uses store (and possibly other services). **No** import of `internal/api`.

- [ ] **Errors**  
  - Domain sentinels only; wrap with `%w`; no logging in service.

## 6. Mastodon API handlers

**Files:**  
- `internal/api/mastodon/statuses.go` (CreateStatusRequest.Poll, pass-through to service)  
- `internal/api/mastodon/polls.go` (new)

### 6.1 Create status with poll

- [ ] **Request**  
  - `CreateStatusRequest` has `Poll` with options, expires_in, multiple, hide_totals.  
  - Handler maps to `CreateWithContentInput.Poll` and does not embed business rules (e.g. max options) in handler; service or validation layer enforces.

- [ ] **Handler pattern**  
  - Parse body â†’ auth if required â†’ call `StatusService.CreateWithContent` â†’ on error `api.HandleError(w, r, err)` â†’ on success write JSON (including poll in response if present).

### 6.2 Polls handler

- [ ] **Struct and constructor**  
  - e.g. `PollsHandler`, `NewPollsHandler(pollService, ...)`. Only services/config; no store.

- [ ] **GET poll**  
  - Route: e.g. `GET /api/v1/polls/:id`. Optional auth. Calls `PollService.GetByID`. Returns `apimodel.Poll`. Uses `api.HandleError` on error, `api.WriteJSON` on success.

- [ ] **POST vote**  
  - Route: e.g. `POST /api/v1/polls/:id/votes`. Requires auth and scope (e.g. `write:statuses`). Body: e.g. `{"choices": [0,1]}`. Calls `PollService.Vote`. Returns updated `apimodel.Poll`. Path param and body validated; invalid input â†’ 422/400 via `api.HandleError`.

- [ ] **Errors**  
  - All errors through `api.HandleError`; no status code mapping in handler. Use `api.ErrUnauthorized` (or equivalent) when auth is required and missing.
